Vocabularies provide the ability to annotate metadata as well as instance data, and define a powerful extensibility point for OData.

Metadata annotations can be used to define additional characteristics or capabilities of a metadata element, such as a service, entity type, property, function, action or parameter. For example, a metadata annotation may define ranges of valid values for a particular property.

Instance annotations can be used to define additional information associated with a particular result, entity, property, or error; for example whether a property is read-only for a particular instance.

Annotations that apply across instances should be specified within the metadata. Where the same annotation is defined at both the metadata and instance level, the instance-level annotation should override the annotation specified at the metadata level.

Some things to keep in mind about vocabularies:

  • A vocabulary is a namespace containing terms where each term is a named metadata extension for an OData service
  • Anyone can define a vocabulary that is specific to a scenario; more commonly used terms can be added to shared vocabularies such as the OData core vocabularies
  • A service should not require a client to interpret annotations
  • Metadata annotations appear in $metadata whereas instance annotations appear in the actual payload

Display and Publication are core OData vocabularies. You can see metadata annotations applying these terms in the metadata of our sample OData feed.

The Display vocabulary has two terms:

  1. DisplayName is used to define a displayable name for a Container, Set, Type or Property
  2. Description is used to provide additional information about a Container, Set, Type or Property

One common use of the Display vocabulary is to annotate feeds with values that can be displayed in a UI for consuming generic OData feeds.

Directories of publications will frequently use the Publication vocabulary. For example, attribution is a common legal requirement when crawling other data sources.

  1. PublisherName provides the name of the publisher for a given publication.
  2. PublisherId provides the ID of the publisher for a given publication.
  3. You can think of Keywords as a search term. For example OData might have these keywords: web, HTML, RESTful, Web API.
  4. Attribution is used to adhere to copyright law. It allows crediting the author of a work which is used or appears someplace else. As an example for Wikipedia data, each table may have a URL that points to the original Wikipedia page from where the data was collected.
  5. Similarly, AttributionUrl is used to follow the copyright law.
  6. DocumentationUrl provides the URL to where the documentation lives.
  7. TermsOfUseUrl will point to any legal terms of use associated with a publication.
  8. PrivacyPolicyUrl will point to a privacy policy document/EULA associated with a publication.
  9. LastModifed shows how fresh the data is. For example in case of Wikipedia it may show the last time the page was crawled.
  10. ImageUrl provides the information about where the icon is for a given publication.
